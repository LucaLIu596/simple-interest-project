# Code of Conduct
This Code of Conduct governs all participants, maintainers, and contributors of the simple-interest shell script repository. Every member joining this open-source project must abide by these standards to maintain a friendly, inclusive, and productive collaborative environment.

## Expected Positive Behavior
- Communicate politely and inclusively, regardless of skill level, background, or experience with Git and shell scripting.
- Accept constructive feedback on code, documentation, and wording without defensive responses.
- Offer patient guidance to new contributors who are unfamiliar with repository workflows or script writing.
- Focus discussion on improving the project rather than personal disagreements.
- Use neutral, respectful language in all issues, pull requests, and comment threads.

## Unacceptable Conduct
- Discriminatory, offensive, insulting, or harassing remarks targeting any individual or group.
- Personal attacks, trolling, or repeated negative provocation in public or private messages.
- Unauthorized disclosure of others’ private personal information.
- Spamming irrelevant links, advertisements, or unrelated content in project discussions.
- Any behavior that makes other contributors feel uncomfortable, threatened, or excluded.

## Reporting Misconduct
If you observe a violation of this code of conduct, send a private message to the repository maintainer on GitHub. All reports will be reviewed confidentially without public disclosure of your identity.

## Enforcement Rules
Repository maintainers hold the authority to remove inappropriate comments, close disruptive pull requests, or restrict contributor access for repeated violations. All enforcement decisions will be supported by clear evidence of the violating behavior.
